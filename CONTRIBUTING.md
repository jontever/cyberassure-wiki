# Contributing to CyberAssure

Thank you for helping build this resource. Contributions of all sizes are welcome.

## How to contribute

1. Fork the repository
2. Create a branch: `git checkout -b feature/your-topic`
3. Add or edit Markdown pages under the relevant section folder
4. Run `npm run dev` locally to preview
5. Submit a Pull Request with a clear description of what you've added

## Content guidelines

- Use clear, accurate, vendor-neutral language where possible
- Cite sources and standards (NIST, ISO 27001, MITRE, CIS, etc.)
- Use diagrams (Mermaid or embedded SVG) to illustrate architectures
- Flag commercial products as examples, not endorsements

## Page structure

Each topic page should include:

- **Overview** — what it is and why it matters
- **Key Concepts** — definitions and principles
- **Architecture Patterns** — how it is applied
- **Attack Vectors & Mitigations** — threat context
- **Implementation Guidance** — practical steps
- **Benchmarks & Standards** — relevant frameworks
- **Further Reading** — links and references
